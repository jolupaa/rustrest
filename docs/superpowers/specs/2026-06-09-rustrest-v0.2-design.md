# RustRest v0.2 — Framework Improvements Design

- **Status:** P0 + P1 implemented (see deviations below); P2 pending
- **Date:** 2026-06-09
- **Author:** jolupa + Claude
- **Branch:** `feat/v0.2-framework-improvements`
- **Version target:** `0.1.0` → `0.2.0` (breaking)

## Context & Goal

`rustrest` is a hand-written Express-style HTTP framework over hyper 1.x + tokio.
The v0.1 core (routing, onion middleware, sync/async handlers, extractors, state,
SSE, WebSockets, static files, panic isolation) is sound. The goal of v0.2 is to
close the gap from "good learning framework" to "feasible alternative for building
real HTTP servers and APIs" by fixing correctness/robustness/security gaps and
adding the batteries an API framework is expected to ship.

## Approved decisions

1. **Cadence:** Implement all **P0** first, checkpoint with the user, then **P1**, then **P2**.
2. **Breaking changes:** Allowed. Bump to `0.2.0`; update examples and tests to match.
3. **Code structure:** Split the monolithic `src/app.rs` into focused modules.
4. **Scope:** Full roadmap — P0 + P1 + P2 (including HTTP/2, radix router, OpenAPI).

## Target module layout

Split `src/app.rs` into focused files under `src/app/`; existing submodules stay.

```
src/app/
  mod.rs        // module wiring + public re-exports
  request.rs    // Request + query/cookie/percent-decode helpers
  response.rs   // Response, BodyKind, IntoResponse
  handler.rs    // Handler/Next/Middleware types, IntoHandler, IntoMiddleware, panic guard
  router.rs     // Router, Route, Segment, parse/match, static files, RouteHandle
  server.rs     // App, ServerConfig, listen/serve/handle/dispatch
  state.rs      // StateStore
  error.rs      // existing — HttpError, IntoHttpError
  extract.rs    // existing — FromRequest + Json/Path/Query/State (+ new extractors in P1)
  middleware.rs // existing — built-in middleware (grows; may become a dir in P2)
  sse.rs        // existing
  websocket.rs  // existing
  tests.rs      // existing — unit tests
```

Guiding principle: each module has one clear purpose, communicates via well-defined
types, and is testable in isolation. Public API remains importable from the crate
root (`rustrest::...`) and from `rustrest::app::...`.

## P0 — production blockers (this phase)

### 1. Binary-safe request body
- `Request.body` (currently `pub String`, UTF-8 lossy) becomes a **private `Bytes`**.
- New accessors:
  - `req.bytes() -> &[u8]` — raw, uncorrupted bytes.
  - `req.text() -> Cow<'_, str>` — lossy UTF-8 view (back-compat for text use).
  - `req.json::<T>()` — unchanged signature, reads from bytes.
- Rationale: binary uploads (images, protobuf) are currently corrupted by `from_utf8_lossy`.

### 2. Response encapsulation
- `status`, `body`, `content_type`, `headers` become **private**.
- Getters added: `res.status() -> u16`, `res.content_type() -> &str`,
  `res.header(name) -> Option<&str>`, `res.headers() -> &HeaderMap`, `res.body_bytes() -> Option<&[u8]>`.
- Internal body representation unified so the `body`/`body_kind` desync (streams report an
  empty `body`) is impossible. Builder methods (`.status()`, `.header()`, `.json()`, …) unchanged.
- Crate-internal access for middleware via `pub(crate)` or methods.

### 3. Server lifecycle: no-panic, Result, graceful shutdown
- `listen` / `serve` return `io::Result<()>` instead of panicking.
- `listen(addr: impl ToSocketAddrs)` (was `&str`).
- Accept-loop errors are **logged and retried**, with a short backoff on `EMFILE`/`ENFILE`;
  a transient accept error never kills the server.
- New: `serve_with_shutdown(listener, shutdown: impl Future<Output = ()>)` and
  `listen_with_shutdown(addr, shutdown)` that stop accepting and drain in-flight connections.

### 4. Server config: limits + timeouts
- `ServerConfig` with builder-style setters on `App`:
  - `app.max_body_size(bytes)` — default 64 KB; **overflow returns `413 Payload Too Large`**
    instead of silently substituting an empty body.
  - `app.request_timeout(Duration)` — whole-request timeout (→ `504`/`408` as appropriate).
  - `app.header_read_timeout(Duration)` — slow-loris protection on headers.
- Defaults preserve current behavior where sensible (64 KB body).

### 5. Client / peer address
- `Request` stores `Option<SocketAddr>`; `req.remote_addr() -> Option<SocketAddr>`.
- Threaded from `listener.accept()` (currently discarded). `X-Forwarded-For` honoring is opt-in (P1).

### 6. Per-route middleware
- Route registration returns a chainable handle:
  ```rust
  app.get("/admin", handler).layer(require_auth).layer(rate_limit);
  ```
- `RouteHandle` borrows the router and targets the just-registered route; return value is
  ignorable when no per-route middleware is needed (current call sites keep compiling-ish;
  they compile since `()`-discard of a must-use-free handle is allowed).

### 7. Routing HTTP-correctness
- **405 Method Not Allowed** with an `Allow` header when the path matches but the method does not.
- **HEAD** auto-served from a matching `GET` route (body stripped) when no explicit HEAD route exists.
- **OPTIONS** auto-answered with `Allow` listing the methods registered for that path
  (unless an explicit OPTIONS route exists). Toggleable.

### 8. Error handler reaches everything
- Built-in `Response::not_found()`, `bad_request()`, and the 404/405 fallbacks carry an
  attached `HttpError`, so a registered `error_handler` (e.g. JSON error envelope) applies to
  404/405/unmatched routes — not only to `Response::from_error`.

### 9. Multi-value request headers
- Request headers stored without collapsing duplicates.
- `req.header(name) -> Option<&str>` (first value, case-insensitive) kept;
  add `req.headers_all(name) -> Vec<&str>`.

### Cross-cutting (P0)
- Version bump to `0.2.0`; update `examples/` and all tests.
- `#![forbid(unsafe_code)]` (no unsafe is used today).

## P1 — API batteries (next phase)

- **Form & multipart bodies:** `req.form::<T>()` (urlencoded) and `multipart/form-data`
  file uploads (`req.multipart()`), behind a feature if it adds heavy deps.
- **Configurable CORS** with automatic preflight (204) and origin allowlist/credentials,
  replacing the hardcoded `*` middleware.
- **Rich cookies + sessions:** `Cookie` builder (`Secure`, `SameSite`, `Max-Age`, `Domain`,
  `Expires`, removal); signed/encrypted cookies; minimal session middleware.
- **`tracing` integration:** structured logging/spans behind a feature; upgrade the
  `request_id`/`tracing` middleware to emit real spans.
- **Test client + public `Request` builder:** `TestClient::new(app).get("/x").send().await`
  and a public `Request::builder()` so downstream crates can unit-test handlers without TCP.
- **TLS via rustls** (feature-gated `tls`): `app.listen_tls(addr, config)`.
- **Static files:** streamed file bodies, `Range` requests, `ETag`/`Last-Modified` +
  conditional GET, `Cache-Control`.
- **More extractors:** `Header<T>`, `Cookie<T>`, `Bytes`, `String`, `Method`, optional/`Result`
  wrappers; typed scalar `Path<u32>`.
- **Compression options:** brotli/deflate in addition to gzip; gzip min-size threshold.

## Implementation deviations (agreed/justified during build)

- **Response encapsulation (P0 §2):** only the `body` field was privatized (the
  one that desynced from `body_kind`); `status`/`content_type`/`headers` remain
  public readable fields because zero-arg getters would collide with the fluent
  builder methods of the same names (`.status(404)`, `.header(k, v)`).
- **gzip min-size threshold (P1):** lives in the new `middleware::compression()`
  (default 1 KB, `compression_with_min_size`); the legacy `gzip()` helper keeps
  its unconditional behavior for back-compat.
- **HTTP/2 (planned for P2):** arrived early in P0 via hyper-util's
  `auto::Builder` (plain and TLS, with ALPN h2 + http/1.1).
- **Typed `Header<T>`/`Cookie<T>` extractors (P1):** name-keyed single-value
  extractors are not expressible without const-generic strings; delivered as
  whole-map `Headers<T>` / `Cookies<T>` instead (serde field names select keys).

## P2 — scale & differentiation (final phase)

- **HTTP/2** (and h2c) behind a feature; auto protocol selection.
- **Radix/trie router** for O(path-length) matching and free 405/Allow.
- **Route introspection:** `app.routes()` listing/printing; trailing-slash policy.
- **Middleware batteries:** rate limiting (needs peer IP), timeout middleware, ETag middleware.
- **OpenAPI generation** from typed extractors/handlers (minimal first cut) + Swagger UI route.
- **WebSocket:** subprotocol negotiation, max-message config, automatic ping/pong keepalive,
  broadcast/channel helper. **SSE:** heartbeat helper + last-event-id resume.
- **Dependency trim:** narrow `tokio`/`hyper` feature sets.

## Testing strategy

- TDD: write/extend tests before each change. Keep `src/app/tests.rs` unit tests and
  `tests/http_integration.rs` real-TCP tests green at every step.
- Each P0 item adds focused tests (e.g. 413 on overflow, 405 + Allow, HEAD from GET,
  binary body round-trip, per-route middleware scoping, graceful shutdown drains).
- `cargo test`, `cargo clippy --all-targets`, `cargo fmt --check` must pass before checkpoint.

## Migration notes (0.1 → 0.2)

- `req.body` field removed → use `req.bytes()` / `req.text()` / `req.json()`.
- `Response` public fields removed → use getters; keep using builder methods to construct.
- `app.listen(addr).await` now returns `io::Result<()>` → `app.listen(addr).await?;`
  (or `.expect(...)` / ignore). Address accepts `impl ToSocketAddrs`.
- Method mismatch now yields 405 (was 404); OPTIONS/HEAD may be auto-answered.
- `error_handler` now also handles 404/405 — verify custom handlers expect this.
